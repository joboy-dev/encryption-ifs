Install go
# Update your package list
sudo apt update

# Install Go
sudo apt install golang-go -y

# Verify the installation
go version

Set go path
Sometimes the system installs Go but doesn't add it to your profile. Add these lines to the end of your ~/.bashrc file:
export GOPATH=$HOME/go
export PATH=$PATH:/usr/local/go/bin:$GOPATH/bin

Update your /etc/hosts file

You need to map the Fabric network's domain names to your local IP address (127.0.0.1).

Open your hosts file with administrative privileges:

Bash
sudo nano /etc/hosts
Add these lines at the bottom of the file:

127.0.0.1   orderer.example.com
127.0.0.1   peer0.org1.example.com
127.0.0.1   peer0.org2.example.com
Save and exit (Ctrl+O, Enter, then Ctrl+X).

Why this is happening now

When you run a command like peer lifecycle chaincode commit or approve, the CLI tool needs to "broadcast" the transaction to the Orderer. Since the Orderer is configured inside Docker as orderer.example.com, the CLI looks for that exact hostname. Without the host file entry, your computer is looking for it on the public internet instead of inside your own machine.

Troubleshooting Check

After updating the file, try to "ping" the orderer to see if it responds:

ping orderer.example.com
If you see replies from 127.0.0.1, the connection issue is resolved.
